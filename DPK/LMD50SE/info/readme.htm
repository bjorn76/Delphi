General

Welcome to LMD-Tools 5 SE-Edition! This package represents a free extract of more than 100 components of our successful LMD-Tools component suite. Against the first release of the SE edition available on the Delphi 5 Companion CD this version is based on the current 5.0 release of LMD-Tools and provides dozens of new features and enhancements. 
No restrictions exist - all components run outside the IDE as well. However against the full release only major components like the ButtonBar, SplitterPanel, Extended Dialog, Datasensitive or TLMDCalendar related controls are missing.

In the \bin directory you'll find a compiled demo of the Megademo project. Check this demo to learn more about the full LMD-Tools version and ways to order it. Please note that all marketing and help materials and the Megademo refer to version 5.0. The LMD-Tools 6 (VCL) release will be available shortly after final release of Delphi 6 release and will contain almost 300 controls. Visit our web site at http://www.lmd.de for more info about this exciting new release. LMD-Tools CLX 1.0 (Kylix and Delphi 6.0 only) is scheduled for July 2001.

Installation
Usually the installer tried to install package files and helpfile automatically (if you did not uncheck the corresponding option). Otherwise check detailed installation instructions in install.htm.

After Installation
After installation you should check the LMD-Tools Introduction-Helpfile to get an overview about the features of LMD-Tools (lmdintro.hlp in the \help directory). After that you should try the demonstration projects (check overview in \demos\delphi\demos.txt) to learn more about the features of LMD-Tools (almost 40 examples are available!). Besides that a project group called lmddemos_se.bpg for simple access to these demoprojects is available can be found in this folder.

If you encounter any problems please feel free to send us email to support@lmd.de or visit the discussion newsgroups on our web site. Please understand that we do not provide general technical support for this free release.

Updating existing projects to LMD-Tools SE 5.04

Follwing informations are important for user who created applications with the LMD-Tools SE package which was made available on the Delphi 5 Companion CD:
 In spite of drasting changes in the unit structure updating your projects to the new version should be rather straight forward: User of previous LMD-Tools releases have to load and save each form of their project. The easiest way to do this is to press F9 (compile) until the project starts. The CBuilder IDE will detect missing unit references and add them automatically. In the worst case additional units must be added manually to the uses clause of an unit (if type or method was moved to another unit, which is not added by the Delphi IDE automatically). If you get an unknown identifier, error simply check the helpfile where the corresponding type/method is now located and add the unit to the uses clause.

Removing LMD-Tools helpfile index from the IDE

On Win9X/ME systems problems may occur with Delphi Help System after LMD-Tools help was integrated (NT and Win2K systems work always fine). If the VCL index is no longer available (and only in this case) uninstall LMD-Tools index files as follows:

Delphi 5/6
Remove :Index lmdtools.cnt line from the Delphi5.cnt (or rather Delphi6.cnt) file. Alternatively you may replace current DelphiX.cnt file with backup file (e.g. named with Delphi5.001).

CBuilder 5
Remove :Index lmdtools.cnt line from the bcb5.cnt file. Alternatively you may replace file with backup file (e.g. named with bcb5.001).

If index files are not visible immediately delete the corresponding *.gid file (e.g. delphi5.gid) to force an immediate update.
