C++ Builder 5

Installation (Alternative 1)
The current CBuilder 5 version of LMD-Tools SE supports one design time package only. For installing the design time packages manually, you have to install the LMD-Tools design-time package

LMD50SE_B5.BPL 		LMD-Tools Special Edition Components 

in the "Install packages..."-dialog.

Important:
Furthermore you have to add the full path (e.g. (BCB)\lib\lmd50se) LMD-Tools DCU/OBJ files to the CBuilder's library path (Library Page/Environment Options in the Tools Menu).
Alternatively the Path could be added to the system path settings. Please note that unlike in Delphi versions LMD-Tools CBuilder library directories are located below the \Lib directory of
your CBuilder installation.

Beyond that the library directory contains all BPK sources, so that you're able to modify package files youself.

How to add the components (Alternative 2)
You may use this alternative to create new package files if you have problems to install the default packages:

  1. Start Borland C++ Builder 5.
  2. Select 'Install Component ...' from the Component- menu.
  3. Decide wether you want to create a new package or use an existing by clicking on the corresponding tab. If you select first alternative you have to specify a name for the new package.
  4. Change the File Type to 'Source file (*.PAS)'
  5. Choose Browse to open the File Open dialog box. Select the directory which contains the LMD-Tools (The \lib\lmd50\main directory below the Borland CBuilder 3 installation directory) and then search for 'LMD_REG.PAS'.
  6. Choose OK to close the browse dialog box. Choose OK to close the 'Install Component...' dialog box.

The library will be rebuilt. Please ignore the warnings the compiler generates. After that the LMD-Tools tabs will appear at the end of the current component palette.

Note (!!!)
Please don't mix up the two installation alternatives! If you want to change the installation method you have to remove old installation completely!

How to add helpfile into the IDE
By default the installer tries to install helpfile automatically.

Installing Parts of LMD-Tools

By default the LMD_REG.PAS installation unit installs all available components. You can easily uninstall unwanted parts of LMD-Tools by turning compiler switches on/off in the file lmdctabs.inc.
Currently the components are divided up into main groups as follows (you'll find the specific registration units which are called by LMD_REG.PAS in the brackets):

Default components (lmdregst.pas)
Most basic components like standard buttons, containers, panels or labels. These components should be always installed because they are used in most of LMD-Tools component and property editors. The compiler switch is called LMDDEFAULTCP and is
activated by default.

Extended components (lmdregex.pas)
More specific components like TLMDClock, TLMDLabelFill, Splitter controls and more.
The compiler switch is called LMDEXTENDEDCP and is activated by default.

Dialog components (lmdregdg.pas)
All dialog components of LMD-Tools like TLMDAboutDlg, TLMDTipDlg and more.
The compiler switch is called LMDDIALOGCP and is activated by default.

Edit components (lmdregec.pas)
The new edit controls and all listbox/combobox descendants are located here
(from version 4.0R3 on). The compiler switch is called LMDEDITCP and is
activated by default.

System components (lmdregsy.pas)
System components like TLMDShapeHint, TLMDFormStyler and more.
The compiler switch is called LMDSYSTEMCP and is activated by default.

Multimedia components (lmdregmm.pas)
Multimedia components like TLMDWaveComp, all ShapeControls or TLMD3DEffect buttons.
The compiler switch is called LMDMMCP and is activated by default.

Uninstalling LMD-Tools

To remove LMD-Tools SE from your system follow the directions below.

C++ Builder 5

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD-Tools packages from the Packages list and click the "Remove" button.
  • After that remove the (BCB) \lib\lmd50\main or (BCB) \lib\lmd50\free directories from the global library search path in "Tools|Environment Options", Tab Library.

Delphi 5 and 6

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD-Tools packages from the Packages list and click the "Remove" button.


Delphi 5

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD-Tools packages from the Packages list and click the "Remove" button.

 

Removing LMD-Tools files from your hard disk
After uninstalling the components from the CBuilder IDE/Delphi you may remove the component files physically.

Windows NT 3.51
Run the uninstaller app (unwise.exe) located in the LMD-Tools installation directory and follow the instructions.

Windows 95 / Windows NT 4.0 or higher
Open the Add/Remove Programs icon from the Control Panel and double click the "LMD-Tools V5.X" entry.

All the DCUs and backup files have to be removed manually from the LMD-Tools' uninstallation directory. As a rule the installation directory won't be deleted during the unstallation because such files will be generated while you work on the Tools but - for safety's sake - won't be deleted automatically by the uninstaller.
The 32bit-versions will create the key HKEY_CURRENT_USER\SOFTWARE\LMD Innovative\LMD-Tools in the current registry.These entries can be deleted if they're not longer required.
