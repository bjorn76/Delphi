Uninstalling LMD-Tools

To remove LMD-Tools from your system completely follow the directions below.

C++ Builder 5 and 6

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD-Tools packages from the Packages list and click the "Remove" button.
  • After that remove all LMD-Tools paths from the global library search path (Library Page/Environment Options in the Tools Menu).
  • Start OpenHelp tool and remove any LMD-Tools helpfiles (Menu Help|Customize..., select Tab "Link" then).

Delphi 5 to  7

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD-Tools packages from the Packages list and click the "Remove" button.
  • After that remove all LMD-Tools paths from the global library search path (Library Page/Environment Options in the Tools Menu).
  • Start OpenHelp tool and remove any LMD-Tools helpfiles (Menu Help|Customize..., select Tab "Link" then).


Removing LMD-Tools files completely from your hard disk
After uninstalling the components from the CBuilder IDE/Delphi you may remove the component files physically.

  • Open the Add/Remove Programs icon from the Control Panel and double click the "LMD [..]" entry.
  • All the DCUs/object and backup files have to be removed manually from the LMD-Tools' uninstallation directory. As a rule the installation directory won't be deleted during the unstallation because such files will be generated while you work on the Tools but - for safety's sake - won't be deleted automatically by the uninstaller. Alternative: Use the Custom Uninstall method in Uninstaller application and select always all options - this will remove complete paths including generated files. We recommend to use the second way.
  • Remove all dcllmd*.* and lmd*.* files from the \projects\bpl (and  \projects\lib in CBuilder) folders and below your CBuilder/Delphi installation folder. If you had chosen to install runtime files into system directory, remove all lmd*.bpl packages from the System directory of your Windows installation. To be sure search harddisk for these files and remove them all. Do not remove license files or registration utility from the system directory.
  • Be sure that you did not compile LMD-Tools files into a custom package!
  • Possible registry keys can be found under:
    HKEY_LOCAL_MACHINE\SOFTWARE\LMD Innovative\LMD-Tools 2003
    Remove them if you don't want to use LMD-Tools any longer.

C++ Builder 5

The Installer supports automatic installation of packages. If you want to install package manually, follow instructions below:

Manual Installation
LMD-Tools SE releases support one design time package only (unlike e.g. the LMD-Tools 7 full version). For installing the design time package manually, you have to install the LMD-Tools design-time package

LMD70SE_B5.BPL 		LMD-Tools Special Edition Components 

in the "Install packages..."-dialog.

Important:
Furthermore you have to add the full path (e.g. $(BCB)\lib\lmd70) of LMD-Tools DCU/OBJ files to the CBuilder's library path (Library Page/Environment Options in the Tools Menu).
Alternatively the Path could be added to the system path settings. Please note that unlike in Delphi versions LMD-Tools CBuilder library directories are located below the \Lib directory of your CBuilder installation.

How to add helpfile into the IDE
By default the installer tries to install helpfile automatically. To install the LMD-Tools Helpfile manually use Openhelp (Menu Help|Customize).
