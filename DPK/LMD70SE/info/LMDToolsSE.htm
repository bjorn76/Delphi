 Welcome to LMD-Tools 7 Special Edition!

This package represents a free extract of almost 100 components of our successful LMD-Tools component suite. Against previous releases this version is based on the current 7.0 release of LMD-Tools and provides dozens of new features and enhancements. 
No restrictions exist - all components run outside the IDE as well and can be used royality free in your applications.

Limitations against commercial versions are among others:

  • Major components like the ButtonBar, SplitterPanel, Extended Dialog, Datasensitive or TLMDCalendar related controls are missing.
  • Several enhanced editors including new designtime settings editor in V7 are missing.
  • No runtime packages are available.
  • Availables demos are based on an extract of LMD-Tools 4.0 demonstration projects
  • No bonus packages are included

For complete feature list of latest LMD-Tools 7 check our web site. A complete list of all controls is available in the Components Overview.

Installation of LMD-Tools SE version
Usually the installer tries to install package files and helpfile automatically (if you did not uncheck the corresponding option). Otherwise check detailed installation instructions in Installation notes.

After Installation
After installation you should check the LMD-Tools Helpfile to get an overview about the features of LMD-Tools (lmdtools.hlp in the \help directory). After that you should try out demonstration projects (check overview in \demos\delphi\demos.txt) to learn more about the features of LMD-Tools (almost 40 examples are available!). Besides that a project group called lmddemos_se.bpg for simple access to those demoprojects can be found in this folder.
Please also note that About-Dialog of any LMD-Tools control allows enabling/disabling of several global editors.

If you encounter any problems please feel free to visit the discussion newsgroups on our web site or newsserver. Please understand that we do not provide general technical support for this free release.

Location of support files
 If helpfiles were installed automatically, you can use them in the IDE as known from the Delphi/CBuilder helpfiles. Otherwise all helpfiles can be found in the \help folder below the LMD-Tools installation directory. All information files can be found in the \info directory. Interface files can be found in the \source directory.

Location of demos
Demo projects can be found in the \demos folder below the installation directory. Please review demos.txt (Delphi versions only) for more info about included demonstration projects.

 
 Using different Language Resource Files
Translated Resourcestrings can be found in source\strings folder. To change current language simply copy unit file of desired language to library file and recompile your application. Feel free to send us modified resource units for other languages!
Important Note for C++ Builder user
Speed up compile time by including the LMDSEMAX.H header file into one of your projects file.
If you get strange "Bitmap Invalid" exceptions with SpeedButtons or similar, problem is probably caused by use of Codeguard. Check troubleshouting document for more info.
 

Removing LMD-Tools helpfile index from the IDE

On Win9X/ME systems problems may occur with Delphi Help System after LMD-Tools help was integrated (NT and Win2K systems work always fine). If the VCL index is no longer available (and only in this case) uninstall LMD-Tools index files as follows:

In Delphi/CBuilder Help menu select "Customize..." and remove in OpenHelp tool LMD-Tools index files.

To remove index files manually follow these steps:

Delphi 5-7
Remove :Index lmdtools.cnt line from the Delphi5.cnt (or rather Delphi6.cnt/Delphi7.cnt) file. Alternatively you may replace current DelphiX.cnt file with backup file (e.g. named with Delphi5.001).

CBuilder 5/6
Remove :Index lmdtools.cnt line from the bcb5.cnt file. Alternatively you may replace file with backup file (e.g. named with bcb5.001).

If index files are not visible immediately delete the corresponding *.gid file (e.g. delphi5.gid) to force an immediate update.



Updating existing projects to LMD-Tools 7

Unit changes
Several units were reorganized and removed from the packages. So you might encounter following problems when trying to compile an existing project:

  1. File not found errors for certain units
    Following units are no longer required in LMD-Tools:
    lmdbredt.pas, lmdbtn.pas, lmdcctrl.pas, lmdcombo.pas, lmdcompo.pas, LMDComponent.pas, LMDCtrl.pas, lmdctrlA.pas, lmddbctA.pas, lmddbctr.pas, lmddlgS.pas, lmdeditb.pas, lmdeditc.pas, LMDEditM.pas, lmdextcA.pas, lmdextcS.pas, LMDGrapA.pas, lmdgroup.pas, LMDMemoC.pas, lmdmmS.pas, lmdnonvA.pas, lmdnonvS.pas, lmdnwgui.pas, lmdrctrl.pas, lmdrectutils.pas, lmdshapA.pas, LMDStdCA.pas, lmdstdcS.pas (for CBuilder user: .hpp/.cpp extension)

     Solution:
    If compiler complains about these files simply remove the unit from the uses clause. This is the recommended way.
    Alternatively     install the units from the     source\dummies (Delphi only, CBuilder user must always remove references). These units are actually empty but make compiler happy.
     
  2. Unknown Identifier error (Components in forms)
    New or different units are required for a control.

    Solution:
    Simply load the affected form and save it again. Delphi/CBuilder will automatically add required units to the uses clause. T    he simplest way is to press F9 (compile) until the project starts.  The IDE will detect missing unit references and add them automatically.
     
  3. Unknown Identifier error (types)
    In "worst" case additional units must be added manually to the uses clause of an unit (e.g. when type or method was moved to another unit, which is not added by the Delphi IDE automatically).
    If you get an unknown identifier, error simply check the helpfile or search the interface/source files to find out where the affected type/method is defined and add this unit to the uses clause.
    For example: Many button related types are now defined in     LMDButtonBase.pas unit instead of LMDGraph.pas. For example if you have changed in your code the style of a button, it is very likely that you have to add LMDButtonBase to the uses clause.

The mentioned issues deal with unit names only. Like always we tried to keep compatibility as far as possible (concerning properties and objects).

Other changes
The style for TLMDCustomButton and TLMDCustomSpeedButton (and controls using internal SpeedButtons) is now specified by the ButtonStyle property.
In addition the ubsWinXP, ubsWinXPCombo, ubsWinXPTool and ubsAutoDetect styles were removed, because CtlXP property (which is activated by default) is now supported by buttons. In most cases you need not to take care of this change, because we handle conversion automatically. 
Exception: If you selected previously a style like ubsDelphi, the speedbutton may now be shown in Themes look (on Windows XP). If this is an unwanted feature, simply disable CTLXP property (either in object inspector or via component context menu). Note: This is a change against 7.0 Preview, where SpeedButtons had by default disabled CTLXP property.
